Android Edge-to-Edge + BottomChrome 实战规则

这份文档专门记录一次真实修复过程里的关键经验：保留 Android 推荐的 enableEdgeToEdge()，同时让 Mini Player、Bottom Tab 和系统导航栏不再互相重叠。

一句话原则

底部固定控件不能简单贴屏幕底部；必须把 App 内容栈、系统导航栏安全区、动画裁剪视口 分开建模。

为什么会出问题 多轮修复复盘 正确模型 落地规则 参考代码 验收清单 反模式

1. 为什么会出问题

Android 现在推荐 edge-to-edge，尤其 target SDK 35+ 后更应该按 edge-to-edge 思路处理系统栏。 但 edge-to-edge 的含义是：系统栏可以覆盖在 App 内容之上，App 要自己决定哪些内容可以画到系统栏后面，哪些内容必须避开。

错误根因不是 enableEdgeToEdge()，而是：底部固定控件没有正确消费 WindowInsets.navigationBars，并且用 offset 隐藏 Tab 时没有裁剪掉已经移出 App 底部区域的内容。

这类问题会在三种页面里表现不同：

页面类型 正确效果 常见错误
一级页 Mini Player + Bottom Tab 都显示，Tab 在系统导航栏上方。 Bottom Tab 和 Android 三键导航重叠。
二级页，有 Mini Player 只露出 Mini Player，Tab 被隐藏并裁剪。 Tab 被推下去后仍画到系统导航栏区域。
二级页，无 Mini Player 整个 BottomChrome 隐藏，底部只剩系统导航栏。 Mini Player 或 Tab 残留在底部。

2. 本次多轮修复复盘：真正值得带走的经验

这次问题不是一次修改解决的。真正有价值的是每一轮都暴露了一个更深层的边界条件， 最后才形成“安全区 + 内容栈 + 裁剪视口”的完整模型。

轮次 当时看到的问题 容易做出的错误结论 最终学到的规则
第一轮 App 没有适配顶部状态栏和底部系统导航栏。 把系统栏隐藏掉，或者让窗口重新 fit system windows。 系统栏应该保持可见；音乐 App 不是全屏游戏。
第二轮 用 decorFitsSystemWindows(true) 可以止血。 这就是最优解。 长期方案仍应保留 enableEdgeToEdge()，因为 Android 15+ 推荐 edge-to-edge。
第三轮 无 Mini Player 的二级页仍可能有底部残留。 再多加一点底部 padding。 隐藏态位移必须包含 App 内容栈和系统导航栏高度。
第四轮 有 Mini Player 的二级页，Bottom Tab 被推下去后仍画到系统导航区。 继续调高度或调 offset。 用 offset 移出视口的内容必须被父容器 clipToBounds() 裁剪。
举一反三： 只要一个 UI 使用 offset、translation、滑入滑出动画来隐藏控件， 都要同时检查三个问题：动画距离是否覆盖安全区、父容器是否裁剪、隐藏后的控件是否还能响应点击。

3. 正确模型

把底部区域拆成三层，问题会清楚很多：

页面内容：列表、标题、按钮，需要避开底部 Chrome 和系统导航栏。
BottomChrome 内容栈：Mini Player + Bottom Tab，只允许在自己的视口里移动。
系统导航栏安全区：读取 WindowInsets.navigationBars，只铺底色，不放可点击控件。
关键判断： 如果一个控件通过 offset 被推出可视区域，它的父容器通常要加 clipToBounds()。否则控件虽然“位置下去了”，但仍然会被绘制出来。

4. 落地规则

规则 原因
保留 enableEdgeToEdge() 符合 Android 当前方向，不用回退到旧的全局 fitSystemWindows 思路。
不要隐藏系统栏 音乐 App 不是全屏游戏，隐藏系统栏会让用户失去系统导航反馈。
页面内容加 statusBarsPadding() 与 navigationBarsPadding() 滚动内容不要被状态栏、导航栏压住。
BottomChrome 显式读取 WindowInsets.navigationBars 底部固定控件需要知道真实系统导航栏高度。
Mini Player + Bottom Tab 仍然只用一个整体动画 避免两个控件分别运动，产生不协调的 UI。
给可动画内容栈外层加 clipToBounds() 普通二级页隐藏 Tab 时，Tab 不能继续画到系统导航栏区域。

5. 参考代码

Activity：保留 edge-to-edge，不隐藏系统栏

override fun onCreate(savedInstanceState: Bundle?) {
    enableEdgeToEdge(
        statusBarStyle = SystemBarStyle.light(
            scrim = android.graphics.Color.TRANSPARENT,
            darkScrim = android.graphics.Color.TRANSPARENT,
        ),
        navigationBarStyle = SystemBarStyle.light(
            scrim = android.graphics.Color.TRANSPARENT,
            darkScrim = android.graphics.Color.TRANSPARENT,
        ),
    )
    super.onCreate(savedInstanceState)
    WindowCompat.setDecorFitsSystemWindows(window, false)
    WindowInsetsControllerCompat(window, window.decorView).apply {
        isAppearanceLightStatusBars = true
        isAppearanceLightNavigationBars = true
    }
    setContent {
        App()
    }
}

BottomChrome：安全区、动画栈、裁剪视口分离

@Composable
private fun BottomChrome(
    placement: BottomChromePlacement,
    modifier: Modifier = Modifier,
) {
    val navigationBarHeight: Dp = with(LocalDensity.current) {
        WindowInsets.navigationBars.getBottom(density = this).toDp()
    }
    val stackHeight: Dp = scaledDp(
        MusicDimens.MiniPlayerHeight + MusicDimens.BottomNavHeight,
    )
    val stackOffset: Dp by animateDpAsState(
        targetValue = when (placement) {
            BottomChromePlacement.TopLevel -> 0.dp
            BottomChromePlacement.MiniPlayerOnly -> scaledDp(MusicDimens.BottomNavHeight)
            BottomChromePlacement.Hidden -> stackHeight + navigationBarHeight
        },
        animationSpec = tween(durationMillis = 780, easing = FastOutSlowInEasing),
        label = "BottomChromeOffset",
    )
    Box(
        modifier = modifier
            .fillMaxWidth()
            .height(stackHeight + navigationBarHeight),
    ) {
        Box(
            modifier = Modifier
                .align(Alignment.BottomCenter)
                .fillMaxWidth()
                .height(navigationBarHeight)
                .background(MusicColors.Paper),
        )
        Box(
            modifier = Modifier
                .fillMaxWidth()
                .height(stackHeight)
                .clipToBounds()
                .align(Alignment.TopCenter),
        ) {
            Box(
                modifier = Modifier
                    .fillMaxWidth()
                    .height(stackHeight)
                    .offset(y = stackOffset),
            ) {
                MiniPlayer(modifier = Modifier.align(Alignment.TopCenter))
                BottomNavigation(modifier = Modifier.align(Alignment.BottomCenter))
            }
        }
    }
}

6. 验收清单

每次改 BottomChrome，都必须同时验收一级页、二级页有 Mini Player、二级页无 Mini Player 三种状态。
检查项 通过标准
一级页 Bottom Tab 完整显示在系统导航栏上方，不重叠。
专辑页/歌手页 只显示 Mini Player；Bottom Tab 不露到系统导航栏区域。
设置页/播放页 Mini Player 和 Bottom Tab 都不显示，底部没有残影。
动画过程 Mini Player 和 Bottom Tab 作为整体移动，不分裂、不跳动。
内容滚动 列表最后一项不会被 Mini Player、Tab 或系统导航栏挡住。

7. 反模式

反模式 1：为了不重叠直接改成 WindowCompat.setDecorFitsSystemWindows(window, true)。 这能短期止血，但不是 Android 15+ 推荐方向。
反模式 2：只给根 Box 加 navigationBarsPadding()。 如果内部内容用 offset 滑出，仍可能画到系统导航栏区域。
反模式 3：Mini Player 和 Bottom Tab 分别做动画。 这样很容易出现两个控件不同步、间隙变化、隐藏状态误触。
