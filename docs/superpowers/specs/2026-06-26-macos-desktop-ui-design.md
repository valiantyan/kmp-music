# macOS 桌面版 UI 1:1 复刻设计

## 背景

KMP Music 当前 macOS Desktop 入口仍以移动端窗口和共享手机 UI 为主，`DesktopMain.kt` 默认窗口为 `430.dp x 930.dp`。项目已经提供桌面版 UI 文档：

`prototypes/kmp-music-desktop-uiux/index.html`

本次设计以该 HTML 为 macOS 桌面版唯一视觉标准，目标是在真实 KMP Desktop App 中按 1:1 复刻其桌面体验，同时保持 Android/iOS 手机 UI 现状不变。

## 目标

- macOS Desktop 使用桌面专用 Compose Shell，不再显示移动端窄屏外壳。
- 视觉以 `index.html` 为准，复刻顶部 macOS 标题栏、左侧导航 rail、中部 workspace、底部播放器，以及页面内表格、卡片、分段控件、设置双栏等视觉节奏。
- 继续复用现有 `MusicAppController`、domain、repository、播放、收藏、搜索、扫描、导航状态，不复制业务逻辑。
- 主界面和二级页全部桌面化。HTML 已覆盖的本地音乐/首页、收藏、我的、设置需严格贴近原型；HTML 未展开的搜索、播放详情、专辑、歌手、本地来源、登录等页面使用同一套桌面视觉语言补齐。
- 移动端 `MusicApp`、Android/iOS 入口和手机布局不作为本次改造目标。

## 非目标

- 不使用 WebView 包装 HTML 原型。
- 不重写播放、扫描、收藏、搜索、持久化等业务能力。
- 不修改 Android/iOS 手机 UI 的信息架构和视觉密度。
- 不把桌面布局分支塞进现有手机页面，避免让移动端承担桌面复杂度。
- 不在本次设计中引入在线音乐、会员、歌词等 PRD 外能力。

## 视觉标准

桌面版视觉直接映射 HTML 原型中的结构和 token：

- 最小窗口参考：`1120 x 760`。
- App grid：顶部 `42px`，左侧 rail `88px`，底部播放器 `96px`，中部 workspace 自适应。
- 背景：浅灰页面底色叠加轻量薄荷色径向高光。
- 主色：`#17b59e`，深主色 `#0fa890`，柔和主色 `#e8f8f5`。
- 文本：主文本接近 `#07090c`，次级文本接近 `#7f8a99`。
- 分割线：接近 `#e7ecf0`。
- 播放中和收藏强调：红色接近 `#ef3f42`。
- 圆角：导航项 `14px`，统计卡 `14px`，内容列表 `16px`，设置卡 `16px`，专辑封面 `12px`。
- 桌面字体使用系统 sans-serif，字号和行高按 HTML 原型映射到 Compose dp/sp，不沿用移动端视觉缩放。

## 信息架构

Desktop 左侧 rail 常驻：

- 首页
- 收藏
- 我的
- 设置

其中“设置”在 Desktop 可常驻左侧 rail，但仍然是二级/管理页面语义，与 PRD 的桌面规则一致。

顶部全局搜索入口进入桌面搜索页。底部播放器全局常驻，承载当前歌曲、播放控制、进度和队列入口。

## 架构设计

新增 Desktop 专用 UI 入口：

- `DesktopMain.kt` 将窗口尺寸改为桌面默认尺寸，并调用 `DesktopMusicApp(controller)`。
- `DesktopMusicApp` 位于 Desktop 可用的 Compose 源集中，负责桌面 Shell 和页面路由。
- 现有 `MusicApp` 继续作为移动端共享 UI 入口，不被桌面 1:1 复刻牵动。

建议模块边界：

- `feature/desktop/DesktopMusicApp.kt`：桌面根 Shell、顶部栏、左侧 rail、workspace、底部播放器、overlay 接入。
- `feature/desktop/DesktopMusicScreens.kt`：桌面主页面和二级页面路由。
- `feature/desktop/DesktopMusicComponents.kt`：桌面表格行、统计卡、专辑网格、分段控件、设置行、桌面按钮等组件。
- `feature/desktop/DesktopMusicTokens.kt`：桌面专用颜色、尺寸、间距、字号 token。

如果 Compose Multiplatform 源集访问需要保持 `commonMain` 兼容，可把 Desktop UI 组件放入 `commonMain` 的 `feature/desktop` 包，但只从 `desktopMain` 入口调用。该方式仍然保持移动端不引用这些组件。

## 页面设计

### 桌面 Shell

Shell 使用 HTML 的三段式布局：

- 顶部栏：左侧 macOS traffic lights，居中 `KMP Music` 标题，右侧圆角全局搜索框。
- 左侧 rail：品牌音符标识，首页/收藏/我的/设置四个 64x64 导航项，当前项显示薄荷色背景和主色图标文字。
- Workspace：从 `grid-column: 2` 开始，页面滚动只发生在 workspace 内部，内边距匹配 HTML 的 `34px clamp(34px, 4vw, 68px) 30px`。
- 底部播放器：跨越全宽，三列布局 `310px / 1fr / 330px`，包含当前歌曲、播放模式、上一首、播放/暂停、下一首、进度、音量视觉和队列入口。

### 首页/本地音乐

HTML 中首页实际以“本地音乐”为首屏，桌面版按此复刻：

- 标题为“本地音乐”，副标题展示真实曲库统计。
- 右侧提供“重新扫描”和更多按钮。
- 三个统计卡：歌曲、专辑、歌手。
- 工具条：播放全部、排序控件。
- 歌曲列表使用桌面表格，列包含序号、标题、歌手、专辑、时长、添加时间、更多。
- 当前播放行使用浅薄荷色横向高亮和左侧主色竖条，歌曲标题使用播放红色。
- 底部展示最近播放的专辑网格。

### 收藏

- 标题为“收藏”，副标题为“喜欢的音乐都在这里”。
- 顶部右侧使用歌曲/专辑/歌手分段控件。
- 统计卡展示收藏歌曲、收藏专辑、收藏歌手。
- 歌曲分段使用表格，列包含收藏心形、序号、标题、歌手、专辑、时长、收藏时间、更多。
- 专辑和歌手分段使用同一桌面视觉语言，不沿用手机窄屏列表。

### 我的

- 顶部为“我的 / 本地资料与同步状态”。
- 登录资料区使用 HTML 的横向 profile panel：头像、登录标题、同步说明、立即登录按钮。
- 四个宽统计卡：本地专辑、歌手、收藏、最近播放。
- 内容列表包含我的收藏、常听歌手、本地文件夹、同步与备份。
- 常听歌手在同一行内横向展示头像和简短统计。

### 设置

- 标题为“设置”，副标题为“播放、扫描与显示偏好”。
- 设置页使用双栏布局：左侧设置分类 nav，右侧设置卡。
- 分类包含通用、播放、本地音乐、外观、同步、关于。
- 右侧通用卡复刻 HTML 的设置行：外观模式、启动时恢复播放、关闭窗口时继续播放、本地文件夹、重新扫描、本地缓存、关于。
- 暂无真实状态的开关先保持 UI 可见，但不得伪造已实现的业务效果；没有业务能力的操作可以保留占位行为或进入已有页面。

### 搜索

- 从顶部全局搜索框进入。
- 使用桌面 page header、搜索输入、scope 分段控件和结果表格。
- 搜索结果表格复用本地音乐表格组件，支持播放、打开专辑/歌手、更多操作。

### 播放详情

- 从底部播放器进入。
- 使用桌面沉浸式内容区，但仍保留桌面 Shell 的顶部、左 rail 和底部播放器。
- 内容可采用左右分栏：左侧大封面和主控制，右侧歌曲信息、播放模式、队列摘要。
- 控制行为复用现有播放状态和 use case。

### 专辑与歌手详情

- 使用桌面 header 区展示封面/头像、标题、元信息、播放全部按钮。
- 主体使用歌曲表格和专辑网格。
- 当前播放状态和收藏状态与全局同步。

### 本地来源与登录

- 本地来源使用桌面设置/管理类页面语言，列表展示来源、扫描问题和重新扫描入口。
- 登录页使用桌面卡片，保持邮箱登录占位能力，不扩大认证范围。

## 数据流与交互

- 所有页面读取 `MusicAppUiState`。
- 导航、播放、收藏、搜索、扫描、队列、更多菜单继续调用 `MusicAppController`。
- Desktop rail 的首页/收藏/我的调用 `navigateToRoot`。
- Desktop rail 的设置调用 `navigateToSecondary(SecondaryScreen.Settings)`。
- 顶部搜索调用 `openSearch`，输入变化调用 `setSearchQuery`。
- 底部播放器调用 `togglePlayback`、`moveTrack`、`cyclePlaybackMode`、`seekTo`、`openQueue`、`openPlayer`。
- 现有 overlay 行为，包括队列、更多菜单、权限设置、清缓存确认，需要在桌面 Shell 中仍可访问。

## 错误与空状态

- 无本地歌曲时，首页表格区域展示空状态和重新扫描入口。
- 无收藏歌曲/专辑/歌手时，对应分段展示空状态，不出现空白表格。
- 当前播放为空时，底部播放器展示默认状态和不可用控制反馈。
- 扫描错误、权限问题和播放错误沿用现有状态与文案，但使用桌面卡片或对话框呈现。

## 测试与验证

- 至少运行 `./gradlew :composeApp:desktopTest`。
- UI 大改后运行 Desktop 编译相关任务；如任务名不确定，先查 `./gradlew :composeApp:tasks`。
- 改动导航、播放、收藏、搜索状态时，补充或更新 `MusicAppControllerTest`。
- 启动 macOS Desktop App，对照 HTML 原型检查：
  - 默认窗口是否为桌面尺寸。
  - 顶部栏、左 rail、workspace、底部播放器是否按原型分区。
  - 首页/收藏/我的/设置是否匹配 HTML 的密度、色彩、表格和卡片结构。
  - 当前播放红色、高亮行、收藏心形、底部播放器状态是否与真实状态同步。
  - 搜索、播放详情、专辑、歌手、本地来源、登录等二级页是否保持桌面语言一致。

## 实施顺序

1. 建立 Desktop 专用入口、窗口尺寸、桌面 Shell 和 token。
2. 复刻底部播放器、左 rail、顶部栏，并接入现有 controller。
3. 复刻首页/本地音乐、收藏、我的、设置四个 HTML 已定义页面。
4. 补齐搜索、播放详情、专辑、歌手、本地来源、登录等二级页。
5. 接入 overlay、空状态、错误状态和真实交互。
6. 运行测试和桌面视觉验收。

## 验收标准

- macOS Desktop 默认显示桌面版 UI，而不是 430dp 手机 UI。
- HTML 已定义的四个页面和全局 Shell 在结构、间距、颜色、密度、组件形态上与原型高度一致。
- Android/iOS 手机 UI 不因本次改造发生视觉或导航回归。
- 桌面端现有播放、收藏、搜索、扫描、队列、设置入口仍可用。
- 构建和相关测试通过；无法完成的视觉或运行验证需在交付说明中明确标注。
