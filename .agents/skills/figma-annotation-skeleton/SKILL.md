# Figma 文本骨架标注

把 Figma 页面解析成一张连续的中文文本骨架图，帮助用户复核页面结构、间距、内边距、控件尺寸、颜色、圆角、字号、行高和关键交互图标。重点是“像设计标注一样看得懂”，不是生成代码、SVG 或图片。

## 工作流程

1. 读取用户提供的 Figma URL、node-id 或当前选中节点。
2. 使用 Figma MCP 获取结构、设计上下文和截图；如果用户说“最新设计稿”“当前页面”，必须重新读取，不沿用旧结论。
3. 先识别页面真实纵向结构：状态栏、标题栏、主内容、卡片组、底部固定区等。
4. 把 Figma 的画板、自动布局、内边距、间距、填充色、描边和文字样式转换成中文标注。
5. 输出一张连续文本骨架图，图内直接写设计参数；只在必要时附 XML 控件名或代码控件名。
6. 输出前按“自审清单”检查，不合格就先修正再回复。

## 读取要求

优先调用以下 Figma MCP 工具：

- `get_metadata`：读取节点层级、坐标、尺寸和命名。
- `get_design_context`：读取样式、字体、颜色、间距和组件上下文。
- `get_screenshot`：核对视觉遗漏，尤其是图标、按压态、固定底栏和被截断内容。

如果 Figma 工具不可用，说明无法直接读取设计稿，并基于用户提供的截图或文本做“待核对版”；不要假装已经读取了 Figma。

## 输出格式

使用一段简短头信息加一个文本代码块。代码块必须是一张从上到下连续的页面骨架图，不拆成多个小标题或参数表。

如果用户只要求输出标注图，最终回复只给标注图和必要的简短备注；不要额外解释格式理论。

推荐头信息：

```text
设计稿：页面名称
Figma 节点：node-id
画板尺寸：宽 dp × 高 dp
页面背景色：#xxxxxx
```

骨架图写法：

```text
┌────────────────────────────────────────────┐
│ nav_top_bar                                │
│ 宽 360dp，高 92dp，背景色 #121212          │
│ ┌────────────────────────────────────────┐ │
│ │ title_bar                              │ │
│ │ 高 52dp，底部分割线 #1FFFFFFF，0.33dp   │ │
│ │ 左侧触控区 48dp × 52dp                  │ │
│ │ 标题：18sp，加粗，行高 22dp，#E6FFFFFF  │ │
│ │ 右侧图标 24dp × 24dp                    │ │
│ └────────────────────────────────────────┘ │
└────────────────────────────────────────────┘
                    ↓ 顶部间距 14dp
┌────────────────────────────────────────────┐
│ main_content                               │
│ 宽 360dp，纵向排列，子控件上下间距 12dp     │
│ ┌────────────────────────────────────────┐ │
│ │ 示例卡片                                │ │
│ │ 宽 328dp，高 144dp，圆角 16dp           │ │
│ │ 背景色 #0AFFFFFF，描边 #1AFFFFFF，0.3dp │ │
│ │▒ 顶部内边距 16dp                      ▒│ │
│ │▒ 标题：14sp，加粗，行高 20dp             │ │
│ │▒ 右侧图标 24dp × 24dp                   │ │
│ │▒ 标题与正文上下间距 8dp                ▒│ │
│ │▒ 正文：16sp，行高 26dp，颜色 #71717A    ▒│ │
│ │▒ 文本区域最小高度 80dp                 ▒│ │
│ │▒ 底部内边距 16dp                      ▒│ │
│ └────────────────────────────────────────┘ │
└────────────────────────────────────────────┘
```

## 标注规则

- 用中文完整表达，不写英文缩写。
- 写“上下间距 12dp”，不要写 `gap 12`。
- 写“上下内边距 12dp”，不要写 `py12`。
- 写“背景色 #121212”，不要写 `bg #121212`。
- 写“圆角 16dp”，不要写 `r16`。
- 写“行高 20dp”，不要写 `lh20`。
- 写“加号图标 24dp × 24dp”，不要写 `+24`。
- 写“复制图标 24dp × 24dp”，不要写 `copy24`。
- 写“文本区域最小高度 80dp”，不要写 `minH80`。
- 保留设计复核需要的参数：尺寸、外间距、内边距、文字样式、颜色、圆角、描边、关键图标、固定定位。
- 对 0.3dp、16.3dp、144.6dp 等 Figma 小数值，保留原值；如果需要实现建议，另起一句写“实现时可按项目规范取整”，不要混进骨架图。
- 如果 Figma 中存在父子分组，优先按设计分组输出，不要只按视觉位置拼凑。
- Android 页面可在控件名称旁补充 XML 控件名，例如 `Hashtag card / aigc_detail_hashtag_card`；仅在用户需要代码映射时添加。

## 图标和状态

- 必须把关键交互图标画进骨架图本体里，不能只在文字里描述。
- 顶部关闭、更多、编辑、加号、复制、上传、箭头、底部按钮等都要核对截图。
- 如果用户提供的是点击后截图，要区分默认态和按压态；只在点击时出现的涟漪或圆形背景标为“按压态”，不能当成默认背景。
- 如果一个 24dp 图标位于 40dp 触控区内，必须同时标注“触控区 40dp × 40dp”和“图标 24dp × 24dp”。

## 禁止项

- 不输出 SVG。
- 不生成图片或临时可视化文件。
- 不把内容拆成多个分段说明。
- 不只列参数表。
- 不使用 `gap`、`py`、`px`、`bg`、`stroke`、`r`、`lh`、`minH`、`maxH`、`+24`、`copy24` 等速记。
- 不漏掉视觉上可见的关键图标。
- 不把截图中的按压态、悬停态或临时态误写成默认态。

## 自审清单

回复前逐项检查：

- 是否重新读取了用户当前给出的 Figma 节点。
- 是否是一张连续的文本骨架图。
- 是否所有标注都是中文完整表达。
- 是否画出了关键图标，尤其是右上角加号、复制、编辑和底部按钮。
- 是否标出了页面背景、主容器宽度、卡片尺寸、间距、内边距、文字样式、颜色、圆角和描边。
- 是否避免了用户明确否定过的格式：SVG、流程图、分段说明、参数表、英文速记。
- 是否在拿不准时标注“待 Figma 核对”或“截图推断”，而不是伪装确定。
